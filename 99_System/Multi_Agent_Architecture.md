> **Цель**: снизить расход Opus-токенов в 5–10 раз без потери качества.

## Диагноз текущего состояния

**Что жрёт токены сейчас:**
- Я (Opus 4.7) делаю всё подряд: пишу длинные .md файлы, парсю чужой текст, копирую файлы, форматирую таблицы, ищу в интернете, читаю inbox.
- Каждое такое действие = Opus-токены, хотя 80% этого хватит Haiku.
- Текущее распределение: ~85% Opus, 10% Sonnet, 5% Haiku. **Инверсия от правильного.**

## Целевая архитектура

```
[Роман]
   ↓ голос/текст
[Я — Opus = ОРКЕСТРАТОР]
   ├─ Стратегия и синтез
   ├─ Разговор с Романом
   ├─ Формирование ТЗ для sub-agents
   ├─ Проверка качества результата
   └─ Маршрутизация задач
        │
        ├─→ [Sonnet sub-agents = МОЗГИ ПОПРОЩЕ]
        │     ├─ writer: длинные .md по моему ТЗ
        │     ├─ researcher: поиск в интернете
        │     ├─ coder: код парсера, тесты, ревью
        │     └─ analyst: анализ csv/json, sql
        │
        ├─→ [Haiku sub-agents = РУКИ]
        │     ├─ fileops: cp/mv/rename/массовые правки
        │     ├─ inbox-processor: ежедневный парс 90_Inbox в digest
        │     ├─ formatter: таблицы, списки, markdown
        │     └─ telegram-relay: ответы из бота на простые вопросы
        │
        └─→ [MCP / Connectors = ИНТЕРФЕЙСЫ]
              ├─ GitHub (коммиты в System-A автоматом)
              ├─ Telegram (push-уведомления Роману «готово»)
              ├─ Google Sheets (если решим экспортировать УЛ)
              └─ Cron / scheduled tasks (ежедневный inbox-digest)
```

## Распределение моделей по типам работы

| Тип работы | Модель | Пример |
|---|---|---|
| Разговор с Романом, стратегия, синтез | **Opus (я)** | "Что делаем дальше с тендер-парсером" |
| Длинная техническая запись (5K+ токенов вывода) | Sonnet | Чеклист аудита v0.2, ТЗ на сервис |
| Поиск фактов в интернете | Sonnet | ГосЛог нормативка, цены VPS |
| Код, тесты, рефакторинг | Sonnet | parser.py, классификатор |
| Парсинг чужого текста (inbox, voice transcripts) | Haiku | дайджест 90_Inbox |
| Копирование/перемещение/правки файлов | Haiku | синхронизация outputs → System-A |
| Форматирование таблиц | Haiku | приведение списка контрагентов в md-таблицу |
| Простые вопросы в Telegram-боте /ask | Haiku | "когда у Пантюшева день рождения" |
| Сложные вопросы в боте с контекстом | Sonnet | "разбери вчерашний рейс" |

## Правила маршрутизации (для меня самого)

1. **<2 минуты работы** — делаю сам. Делегирование само по себе стоит токенов на установку контекста.
2. **5+ минут рутины** (длинный .md, поиск, код 50+ строк) — Sonnet sub-agent. Я даю ТЗ и проверяю.
3. **30+ минут или много мелких файлов** — Haiku sub-agent. Я даю чёткие инструкции и проверяю выборочно.
4. **Параллельные задачи без зависимостей** — пускаю несколько sub-agents одновременно.
5. **Высокая ставка** (юридический документ, налоговые риски, деньги) — двойная проверка: Sonnet делает, Opus проверяет.

> Не все MCP есть в реестре Cowork. Проверить через `mcp__mcp-registry__search_mcp_registry`. Если коннектора нет — sub-agent делает через bash + git CLI.

### Skills (повторяемые паттерны)

Skills = пресеты действий, которые я могу применять без переобъяснения каждый раз:

| Skill | Что делает |
|---|---|
| `carrier-card-create` | По ИНН → полная карточка контрагента в УЛ-формате |
| `tender-classify` | По описанию лота → решение «релевантно/нет» + причина |
| `weekly-inbox-review` | Парсит 90_Inbox за неделю, выдаёт дайджест по темам |
| `pinned-facts-update` | Обновляет `Pinned Facts.md` без потери стиля |
| `case-card-create` | Из фотографий + описания делает карточку для `04_Brand/Cases_Pending.md` |

> Создавать через `skill-creator`. Каждый skill — отдельный markdown с инструкцией, на который я ссылаюсь через `Skill: <name>`.

## Применение прямо сейчас (28.04.26)

### MVP парсера тендеров — пример нового workflow

**Старая модель** (которую я применил бы час назад):
- Я (Opus) пишу 500 строк parser.py
- Я (Opus) тестирую
- Я (Opus) пишу README
- ~30K токенов Opus

**Новая модель**:
1. Я (Opus, 2 мин) пишу ТЗ для Sonnet sub-agent: "напиши Playwright-парсер zakupki.gov.ru, такие-то фильтры, такая-то схема SQLite, такой-то формат вывода".
2. Sonnet sub-agent (5 мин) пишет parser.py + tests + README.
3. Я (Opus, 3 мин) проверяю: запускается ли, правильная ли логика фильтров, нет ли явных ошибок.
4. Если ОК — коммитим. Если нет — откидываю Sonnet'у итерацию.
5. **Экономия: ~70% токенов на этой задаче.**
