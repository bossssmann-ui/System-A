# Discovery Roadmap — интервью по каждому направлению

> Собственник дал общий список направлений. Теперь мы должны по каждому провести **короткое интервью** (15–25 минут), и только потом строить под него конкретную инфраструктуру: папки, карточки, метрики, шаблоны, автоматизации.

## Принцип
Не строим "правильную" структуру заранее — слишком часто она не совпадает с реальностью бизнеса. Спрашиваем, слушаем, потом формируем под реальные поля и процессы.

## Формат одного интервью (25 минут макс)

Каждое интервью идёт по одной структуре, чтобы ответы сводились в общий шаблон инфраструктуры. Примерная рамка:

1. **Сейчас как это работает?** Короткое описание текущего положения.
2. **Где больнее всего?** 2–3 конкретных места, где утечка денег/времени/контроля.
3. **Какие сущности учитываются?** Машины, водители, клиенты, сделки, заявки — с какими полями?
4. **Какие события важны?** Инциденты, поступления, закрытия сделок, просрочки — что фиксируем?
5. **Какие метрики собственника?** По каким 3–5 цифрам ты понимаешь, что направление здорово?
6. **Кто что решает?** Зоны ответственности собственника / РОПа / сотрудников.
7. **Какие повторяющиеся проблемы?** — кандидаты в SOP.
8. **Что уже автоматизировано / должно быть автоматизировано?** — кандидаты для Builder Agent.
9. **Что забрать у собственника?** — кандидаты для делегирования/AI.

## Что выходит из одного интервью

После каждого интервью я генерирую:
- обновление `README.md` соответствующей папки с реальной картиной;
- 1–3 шаблона карточек/журналов под конкретные сущности;
- 2–3 задачи в список "effect of presence" (лёгкие шаги уже сейчас);
- 1–2 SOP в очередь;
- список полезных автоматизаций для Builder;
- что стоит запомнить системе (memory entries).

## Вывод после всех шести интервью

Общая карта:
- единая система сущностей (vehicles, drivers, clients, deals, leads, trips, incidents, payments...);
- единая ролевая модель (кто что решает на каждом уровне);
- единый список приоритетов автоматизации;
- единая метрика собственника (one-page dashboard).

## Ритм

- Не больше 1 интервью в день — иначе собственник вываливает всё разом и теряется фокус.
- Операционка первой (потому что она сейчас самое больное).
- Между интервью — использовать результат предыдущего (заполнять карточки, писать SOP).
