# План мобильного капчура — голос/текст → Obsidian

> Личная память собственника — главное узкое место. Писать руками со смартфона неудобно. Цель: голос в Telegram → расшифровка → автоматическая запись в `90_Inbox/` vault.

## Трёхуровневая стратегия

### Уровень 0 — работает СЕГОДНЯ, без инфраструктуры
- Пишешь/диктуешь прямо в этот AI-чат со смартфона: "запиши в inbox: ...".
- Я добавляю строку в `90_Inbox/Quick.md` с датой.
- Запасной канал навсегда.

### Уровень 1 — Obsidian + iCloud (1 час на настройку)
- На Mac: vault в iCloud Drive.
- На iPhone: Obsidian iOS → открыть vault из iCloud.
- На ходу: открыть Obsidian → Daily Note → нажать 🎤 на клавиатуре iOS → продиктовать → сохранить.
- **Работает из коробки. Голос → текст делает сам iOS.** Правки можно сделать позже.

### Уровень 2 — Telegram-бот → vault (1–2 недели на сборку)
Основной рабочий канал в перспективе. Архитектура:

```
iPhone → Telegram (@myinbox_bot)
           │ голосовое / текст / фото
           ▼
       Bot (серверный)
           │ транскрипция (Whisper API)
           ▼
       Файл в vault: 90_Inbox/YYYY-MM-DD.md
           │ синхронизация
           ▼
   Obsidian на Mac + iPhone
   Claude (AI) читает и работает
```

## Развилка: где живёт vault?

От этого зависит, может ли бот напрямую писать в него. iCloud красив, но **не имеет API** — ни один бот не запишет в iCloud-папку снаружи.

### Вариант A — vault в GitHub private repo + Obsidian Git plugin
- **Плюсы:** API есть (бот пишет commits), история всех изменений, работает везде, бесплатно.
- **Минусы:** нужно один раз настроить плагин и ключи.
- **Стоимость:** $0.
- **Кто может писать в vault:** ты, бот, я (Claude).

### Вариант B — vault в iCloud + receiver на Mac
- **Плюсы:** vault остаётся в iCloud, никуда не мигрирует.
- **Минусы:** Mac должен быть включён и онлайн, чтобы бот мог положить запись. Ночью/вне дома сообщения копятся в очереди.
- **Стоимость:** $0, но зависит от "Mac включён".
- **Кто может писать:** ты, Mac-скрипт (после того, как получит сообщение из Telegram).

### Вариант C — vault в Dropbox / Google Drive + WebDAV-плагин
- **Плюсы:** есть API, работает везде.
- **Минусы:** Obsidian iOS не любит Dropbox напрямую, нужны плагины "Remotely Save".
- **Стоимость:** $0 на старте.

## Рекомендация

**Вариант A (GitHub) — лучший баланс надёжности, цены и автоматизации.**

Но это архитектурное решение, и его я один не принимаю. См. вопрос в чате.

## Что делаем в каком порядке

1. **Сегодня–завтра (Уровень 0+1):** настраиваем Obsidian + iCloud на Mac и iPhone. С этого момента иметь vault + капчур на ходу (iOS dictation).
2. **Эта или следующая неделя:** решаем вилку A/B/C и мигрируем vault если нужно.
3. **1–2 недели после миграции:** Builder Agent собирает Telegram-бот. Спека — в `06_Builder/Telegram_Voice_Bot.md`.
