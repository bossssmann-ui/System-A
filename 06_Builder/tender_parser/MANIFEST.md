# Manifest MVP Парсера Тендеров

## Описание проекта

Автоматический парсер публичных тендеров на грузоперевозки и логистику с площадки **zakupki.gov.ru** для компании "Тихоокеанская звезда" (экспедитор, Приморский край).

## Версия

**0.1.0 (MVP)** — апрель 2026

## Структура проекта

```
tender_parser/
├── parser.py              # Точка входа, argparse, управление потоком
├── scraper.py             # Логика Playwright + парсинг BeautifulSoup

├── MANIFEST.md            # Этот файл
└── tenders.db             # SQLite БД (создаётся при первом запуске)
```

## Быстрый старт

1. **Установка зависимостей**:
   ```bash
   pip install -r requirements.txt
   playwright install chromium
   ```

2. **Запуск парсера**:
   ```bash
   python3 parser.py --limit 50
   ```

3. **Результаты**:
   - Консоль: сводка по найденным тендерам, топ-10 по сумме
   - Файл `tenders.db`: SQLite база с историей всех тендеров
   - Файл `parser.log`: детальные логи операций

## Основной функционал

### Что делает парсер

1. Запускает Chromium браузер через Playwright (headless по умолчанию)
2. Навигирует на zakupki.gov.ru/epz/order/extendedsearch/results.html
3. Применяет фильтры:
   - Законы: 44-ФЗ и 223-ФЗ
   - Этап: "Подача заявок"
   - Ключевые слова: грузоперевозки, перевозка, транспорт, экспедиция, доставка, логистика
   - НМЦ от 60 000 рублей
4. Парсит первые N=50 результатов (настраивается через --limit)
5. Исключает тендеры с авиа-доставкой (стоп-фильтр)
6. Сохраняет в SQLite с дедупликацией (UPSERT)
7. Выводит топ-10 новых по сумме

### Обработка ошибок

- **Retry логика**: если страница не загружается, повторяет 3 раза с экспоненциальной задержкой (2, 4, 8 сек)
- **Graceful degradation**: если Playwright не установлен, использует тестовые данные
- **Логирование**: все ошибки логируются в parser.log и stdout

## Параметры командной строки

```bash
python3 parser.py [OPTIONS]

Options:
  --limit N           Количество тендеров (default: 50)
  --headful           Видимое окно браузера (для отладки)
  --db-path PATH      Путь к БД (default: tenders.db)
  --help              Справка
```

## Примеры использования

```bash
# По умолчанию: 50 тендеров, headless, tenders.db
python3 parser.py

# Парсить 100 тендеров
python3 parser.py --limit 100

# Отладка с видимым браузером
python3 parser.py --limit 10 --headful

# Использовать /var/data/archive.db
python3 parser.py --db-path /var/data/archive.db

# Через bash скрипт
./run.sh --limit 50
```

## Архитектура БД

**Таблица `tenders`**:

| Поле | Тип | Описание |
|------|-----|---------|
| id | TEXT (PK) | Реестровый номер тендера |
| title | TEXT | Наименование закупки |
| law | TEXT | Закон: 44-ФЗ или 223-ФЗ |
| customer | TEXT | Организация-заказчик |
| amount | INTEGER | НМЦ в рублях |
| region | TEXT | Регион РФ |
| deadline | TEXT | Дата/время окончания подачи |
| link | TEXT (UNIQUE) | URL карточки на сайте |
| description | TEXT | Краткое описание |
| fetched_at | TEXT | Дата добавления в БД |
| updated_at | TEXT | Дата последнего обновления |

**Индексы**:
- `idx_fetched_at` — для быстрого получения новых
- `idx_amount` — для сортировки по сумме
- `idx_law` — для фильтрации по типу закона

**UPSERT политика**: если тендер уже в БД (по ID), обновляем сумму и дедлайн, но не перезаписываем дату добавления.

## Логирование

Два потока логирования:

1. **stdout** — в реальном времени во время работы
2. **parser.log** — все события (DEBUG, INFO, WARNING, ERROR)

Формат:
```
2026-04-28 10:15:33,456 - root - INFO - Запуск парсера...
```

## Текущие ограничения (известные проблемы)

### 1. Защита от ботов (Cloudflare)

**Проблема**: закупки.gov.ru может блокировать IP или требовать прохождения Cloudflare Check.

**Симптомы**:
- Ошибка 403 Forbidden
- Timeout при загрузке
- Блокировка IP

**Решение в MVP**:
- Используем реальный User-Agent Chrome
- Retry с экспоненциальной задержкой
- headless режим (по умолчанию)

**План для итерации**:
- Использовать прокси-сервис (ProxyMesh, Oxylabs)
- Добавить random delay между запросами
- Запускать с Xvfb на Linux (виртуальный дисплей)
- Увеличить timeout параметры

### 2. Изменение структуры HTML

**Проблема**: закупки.gov.ru часто обновляет интерфейс.

**Симптомы**:
- Парсер находит страницу, но не извлекает данные
- Пустой результат

**Решение**:
- Селекторы написаны гибко (используются class и tag, а не жёсткие xpath)
- Логируются найденные элементы для отладки

**План**:
- Использовать API закупки.gov.ru (если доступен)
- Инспекция HTML и сохранение снимка при запуске

### 3. Фильтры на сайте не полностью реализованы

**Текущее состояние**: фильтры загружаются, но применяются локально (в памяти), а не через UI.

**План**:
- Применить фильтры через URL параметры
- Или: использовать Playwright для клика на элементы UI
