# Структура проекта MVP парсера тендеров

## Файловая структура

```
tender_parser/
├── Python модули (основной код)
│   ├── parser.py              # Точка входа, argparse, управление потоком
│   ├── scraper.py             # Логика Playwright + парсинг BeautifulSoup
│   ├── storage.py             # SQLite слой (CRUD операции)
│   ├── filters.py             # Фильтры, нормализация, валидация
│   └── config.py              # Конфигурация, константы, переменные окружения
│
├── База данных
│   ├── tenders.db             # SQLite (создаётся автоматически)
│   └── schema.sql             # SQL схема (для справки)
│
├── Конфигурация и зависимости
│   ├── requirements.txt        # Python зависимости (pip install -r)
│   └── .env.example           # Пример файла конфигурации
│
├── Скрипты
│   └── run.sh                 # Bash скрипт для запуска
│
├── Документация
│   ├── README.md              # Основная документация (как запустить, FAQ)
│   ├── MANIFEST.md            # Информация о проекте и ограничениях
│   ├── EXAMPLES.md            # Примеры использования (7 сценариев)
│   ├── DEPLOYMENT.md          # Развёртывание на production
│   └── PROJECT_STRUCTURE.md   # Этот файл
│
└── Логирование
    └── parser.log             # Логи выполнения (создаётся при запуске)
```

## Описание каждого файла

### Основной код

#### `parser.py` (4.7 КБ, ~150 строк)
**Точка входа программы.**

Функции:
- `main()` — основной цикл парсера
- Argparse для параметров: `--limit`, `--headful`, `--db-path`
- Инициализирует БД
- Запускает скрейпер
- Выводит результаты в stdout и логирует

Зависимости: `scraper`, `storage`, `config`, `logging`, `argparse`

#### `scraper.py` (15 КБ, ~380 строк)
**Логика скрейпинга и парсинга.**

Классы:
- `ZakupkiScraper` — основной класс для парсинга
  - `_launch_browser()` — запуск Chromium
  - `_navigate_with_retry()` — навигация с retry логикой
  - `_apply_filters()` — применение фильтров на сайте
  - `_parse_results()` — парсинг HTML
  - `_extract_tender_from_row()` — извлечение данных из одной строки
  - `run()` — основной метод

Функции:
- `run_scraper()` — вспомогательная функция
- `get_mock_tenders()` — тестовые данные (используется если Playwright не установлен)

Зависимости: `playwright`, `beautifulsoup4`, `config`, `filters`

#### `storage.py` (5.6 КБ, ~160 строк)
**SQLite слой (работа с БД).**

Классы:
- `TenderDB` — основной класс для работы с БД
  - `_init_schema()` — инициализация схемы (CREATE TABLE)
  - `upsert_tender()` — вставка или обновление тендера
  - `get_recent_new_tenders()` — получение топ-N тендеров
  - `get_tender_count()` — подсчёт всех тендеров

Зависимости: `sqlite3`, `logging`

#### `filters.py` (2.5 КБ, ~90 строк)
**Фильтры и нормализация данных.**

Функции:
- `has_stop_keyword()` — проверка стоп-слов (авиа, воздушные перевозки)
- `normalize_amount()` — парсинг суммы НМЦ из строки
- `normalize_deadline()` — нормализация даты дедлайна
- `normalize_title()` — очистка заголовка

Зависимости: `config`, `logging`

#### `config.py` (1.3 КБ, ~50 строк)
**Конфигурация и константы.**

Константы:
- `HEADLESS` — режим браузера
- `TIMEOUT_MS` — таймаут загрузки страницы
- `RETRY_ATTEMPTS`, `RETRY_DELAY_SEC` — параметры retry
- `DEFAULT_LIMIT` — количество тендеров по умолчанию
- `MIN_NMC_RUB` — минимальная НМЦ
- `USER_AGENT` — User-Agent браузера
- `ZAKUPKI_SEARCH_URL` — URL поиска на закупки.gov.ru
- `SEARCH_KEYWORDS` — ключевые слова для поиска
- `STOP_KEYWORDS` — стоп-слова для исключения
- `DB_PATH` — путь к БД

Зависимости: `dotenv`

## Точки интеграции

### Входные точки
1. **Командная строка**: `python3 parser.py [args]`
2. **Python код**: `from scraper import run_scraper; tenders = run_scraper()`

### Выходные точки
1. **SQLite БД**: `tenders.db` (можно запрашивать напрямую)
2. **stdout**: таблица с результатами
3. **Файл логов**: `parser.log`

## Расширяемость

Парсер разработан с учётом расширяемости:

1. **Добавить новый фильтр**: расширить `filters.py`
2. **Изменить URL**: обновить `config.py`
3. **Добавить уведомления**: новый модуль `notifications.py`
4. **Классификация**: новый модуль `classifier.py` (Claude API)
5. **REST API**: новый модуль `api.py` (Flask/FastAPI)

## Жизненный цикл тендера в системе

```
1. parser.py запускается (по расписанию или вручную)
   ↓
2. scraper.py запускает Playwright браузер
   ↓
3. Навигирует на zakupki.gov.ru и применяет фильтры
   ↓
4. Парсит HTML с помощью BeautifulSoup
   ↓
5. filters.py нормализует данные и проверяет стоп-слова
   ↓
6. storage.py выполняет UPSERT в SQLite
   ↓
7. parser.py выводит результаты в stdout и логи
   ↓
8. Данные сохраняются в tenders.db для дальнейшего анализа
```
