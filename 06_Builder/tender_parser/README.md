# MVP Парсер тендеров Закупки.gov.ru

Автоматический ежедневный парсер публичных тендеров на грузоперевозки и логистику с площадки **zakupki.gov.ru** (ЕИС, законы 44-ФЗ и 223-ФЗ).

Разработан для ООО "Тихоокеанская звезда" (экспедитор, Дальний Восток, Приморский край).

## Возможности

- Парсит результаты поиска закупки.gov.ru с фильтрами:
  - Законы: 44-ФЗ и 223-ФЗ
  - Этап: "Подача заявок"
  - Ключевые слова: грузоперевозки, перевозка, транспорт, экспедиция, доставка, логистика
  - НМЦ от 60 000 руб
- Исключает воздушные перевозки и авиа-доставку (стоп-фильтр)
- Сохраняет результаты в SQLite с дедупликацией по ID
- Выводит топ-10 новых тендеров по сумме
- Логирует во время работы (stdout + parser.log)
- Поддерживает отладку в режиме headful (видимый браузер)

## Установка

### Требования

- Python 3.10+
- pip

### Шаги установки

1. Клонируйте или скопируйте папку проекта:
```bash
cd tender_parser
```

2. Установите зависимости:
```bash
pip install -r requirements.txt
```

3. Установите Chromium для Playwright:
```bash
playwright install chromium
```

## Использование

### Базовый запуск

```bash
python parser.py
```

Парсит 50 тендеров (по умолчанию), сохраняет в `tenders.db`.

### Параметры командной строки

```bash
python parser.py --help
```

- `--limit N` — количество тендеров для парсинга (по умолчанию 50)
- `--headful` — запустить браузер в видимом режиме (для отладки)
- `--db-path PATH` — путь к файлу БД (по умолчанию `tenders.db`)

### Примеры

```bash
# Парсить 100 тендеров в headless режиме
python parser.py --limit 100

# Парсить 10 тендеров с видимым браузером (отладка)
python parser.py --limit 10 --headful

# Использовать свой путь к БД
python parser.py --db-path /var/tender_parser/archive.db
```

## Настройка cron (Linux/macOS)

Чтобы запускать парсер ежедневно в 03:00 UTC:

```bash
0 3 * * * cd /путь/к/tender_parser && /usr/bin/python3 parser.py >> cron.log 2>&1
```

Или через crontab:

```bash
crontab -e
```

Добавить строку:

```
0 3 * * * cd /path/to/tender_parser && /usr/bin/python3 parser.py
```

Проверить установленные cron-задачи:

```bash
crontab -l
```

## Вывод

Программа выводит:

1. **В stdout**:
   - Количество найденных тендеров
   - Количество новых добавленных в БД
   - Топ-10 новых тендеров по сумме (ID, заголовок, НМЦ, срок, ссылка)

2. **В файл `parser.log`**:
   - Подробные логи всех операций (отладка, ошибки, предупреждения)

## Известные ограничения

### Проблема: Cloudflare / антибот блокировка

**Симптомы**: Код ошибки при загрузке страницы, "403 Forbidden", блокировка IP.

**Причина**: закупки.gov.ru может использовать Cloudflare или другие системы защиты от ботов.

**Текущее решение (MVP)**:
- Используется реальный User-Agent Chrome
- Playwright работает с полноценным браузером (не headless mode по умолчанию для тестов)
- Добавлены retry с экспоненциальной задержкой (2, 4, 8 сек)

**План для следующих итераций**:
1. Использовать прокси-сервис (ProxyMesh, Oxylabs и т.д.)
2. Добавить имитацию задержек между запросами (time.sleep)
3. Запускать в headful режиме с отображением (для человеческого взаимодействия)
4. Использовать Xvfb на Linux для виртуального дисплея
5. Увеличить timeout и retry параметры

### Проблема: Структура HTML меняется

**Симптомы**: Парсер не извлекает данные, хотя сайт доступен.

**Причина**: закупки.gov.ru часто обновляет интерфейс, селекторы CSS могут измениться.

**Текущее решение**:
- Парсер использует генерические селекторы и BeautifulSoup
- Логирует найденные элементы для отладки

**План для итерации**:
1. Добавить инспекцию HTML при запуске (сохранить снимок страницы)
2. Обновить селекторы вручную после проверки сайта
3. Использовать API закупки.gov.ru (если доступен)

### Проблема: Фильтры в браузере не применяются

**Текущее состояние (MVP)**: фильтры не полностью реализованы. Программа загружает страницу, но фильтры (закон, этап, ключевые слова) применяются локально в памяти, а не через UI сайта.

**План для итерации**:
1. Изучить форму поиска на закупки.gov.ru (GET параметры или POST)
2. Применить фильтры через параметры URL
3. Или: использовать Playwright для клика по элементам UI и установки фильтров

## Архитектура БД

Таблица `tenders`:

| Поле | Тип | Назначение |
|------|-----|-----------|
| `id` | TEXT (PRIMARY KEY) | Реестровый номер закупки |
| `title` | TEXT | Наименование закупки |
| `law` | TEXT | Тип закона (44-ФЗ / 223-ФЗ) |
| `customer` | TEXT | Организация-заказчик |
| `amount` | INTEGER | НМЦ в рублях |
| `region` | TEXT | Регион |
| `deadline` | TEXT | Дата/время окончания подачи |
| `link` | TEXT (UNIQUE) | URL карточки тендера |
| `description` | TEXT | Краткое описание |
| `fetched_at` | TEXT | Дата первого добавления |
| `updated_at` | TEXT | Дата последнего обновления |

Индексы:
- `idx_fetched_at` — для быстрого получения новых
- `idx_amount` — для сортировки по сумме
- `idx_law` — для фильтрации по закону

UPSERT: если ID уже существует, обновляем сумму и дедлайн, но не переписываем дату добавления.

## Логирование

Логи пишутся в `parser.log` и stdout. Формат:

```
2026-04-28 10:15:33,456 - root - INFO - Запуск парсера с параметрами: limit=50, headful=False, db_path=tenders.db
2026-04-28 10:15:33,457 - root - INFO - БД инициализирована: tenders.db
2026-04-28 10:15:33,500 - root - INFO - Запуск скрейпера...
```

Уровни логирования:
- `DEBUG` — детальная информация (для отладки)
- `INFO` — основные события
- `WARNING` — предупреждения (например, не удалось распарсить поле)
- `ERROR` — ошибки, но работа продолжается
- `CRITICAL` — критические ошибки, парсер останавливается

## Примеры использования в Python

```python
from scraper import run_scraper
from storage import TenderDB

# Запустить скрейпер
tenders = run_scraper(headless=True, limit=50)
print(f"Найдено: {len(tenders)} тендеров")

# Сохранить в БД
db = TenderDB("tenders.db")
for tender in tenders:
    db.upsert_tender(tender)

# Получить топ-10
top = db.get_recent_new_tenders(limit=10)
for t in top:
    print(f"{t['id']}: {t['title']} ({t['amount']} руб)")
```
