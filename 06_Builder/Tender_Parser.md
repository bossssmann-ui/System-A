# Парсер тендеров для "Тихоокеанской звезды"

> Открыто 27.04.26 по запросу Романа (войс 04-23 00:51).
>
> **Цель:** AI-агент, который заходит на тендерные площадки, читает лоты, скачивает документы, фильтрует по нашему профилю (грузоперевозки, спецтехника, ДВ-логистика), складывает в общий ранжированный список.

## Контекст

- Роман обзвонил **агрегаторы** тендеров. Слишком дорого:
  - **B2B-Center**: 75 000 ₽/квартал = 25 000 ₽/мес
  - **Тендерплан**: 170 000 ₽/квартал = 56 700 ₽/мес
  - **Контур.Закупки**: 200 000 ₽/квартал = 66 700 ₽/мес
- Сейчас в тендерах не торгует — стартуем с нуля.
- **Цель Романа (его слова):** "одна программа, которая работает с 2–3 источниками, сравнивает и исключает повторения".
- **Идея решения:** самописный парсер по первичным площадкам (минуя агрегаторов).
- Альтернатива найма человека: дешевле, не устаёт, не уходит в отпуск.

## Целевые источники (от Романа, 27.04.26)

Прямые тендерные площадки, по которым нужно собирать:

1. **Сбер-А** (sberbank-ast.ru)
2. **РТС-тендер** (rts-tender.ru)
3. **Roseltorg** (roseltorg.ru)
4. **B2B-Center** (b2b-center.ru) — это и площадка, и агрегатор
5. **Тендерпро** (tender.pro)

Покрытие: и **44-ФЗ** (госзакупки), и **223-ФЗ** (закупки госкомпаний). География — **не ограничена**, любая по РФ.

## Тематика лотов (фильтр "наш")

Основной нажим — **транспорт / логистика / экспедирование**:

- Внутренние перевозки по РФ
- Импортные перевозки (внешнеторговые)
- Виды транспорта: **море, ЖД, авто** (мультимодальные тоже)
- Типы груза: **контейнеры, генералка, негабарит, тяжеловес, проектные перевозки**

## Что нужно от инструмента

### Функциональность
1. **Поход по списку площадок** (госзакупки, B2B-Center, Сбер-А, РТС-тендер, региональные ДВ — финальный список ниже надо составить).
2. **Поиск лотов** по фильтру (грузоперевозки, экспедирование, перевозка спецтехники, регион ДВ).
3. **Скачивание документов** (ТЗ, протокол, проект договора).
4. **Анализ лота**: подходит ли нам по критериям (бюджет, сроки, требования к перевозчику, нужны ли допуски и т.д.).
5. **Ранжированный список** утром в Telegram или в файл vault.
6. **Опционально**: автоматическая подача заявки (но это сразу поднимает риски — оставим на ручное "ОК").

### Нефункциональные требования
- Бюджет: до **5–10 тыс. ₽/мес** (если выше — невыгодно vs наёмный человек на пару часов в день).
- Время на внедрение: 2–4 недели максимум.
- Не требует постоянного присмотра: запустил утром — отчёт в Telegram днём.

## Анализ вариантов

### Вариант 1. Claude in Chrome (Anthropic, beta)
- **Что это:** официальный браузерный агент от Anthropic, использует Claude как мозг и Chrome как руки.
- **Сценарий:** "зайди на zakupki.gov.ru, найди лоты по слову 'перевозка', скачай первые 10 ТЗ".
- **+ плюсы:**
  - Самый "умный" агент (Claude Opus/Sonnet), хорошо понимает русский, тендерную лексику.
  - Не нужно программировать сценарий — описываешь словами.
  - Видит экран, понимает капчу, интерфейсы.
- **− минусы:**
  - **Beta**, может ломаться/тормозить.
  - Стоимость зависит от модели — на мощных моделях $5–15 за длинную сессию (агент видит экран = много токенов).
  - Один агент = один Chrome. Параллелить дороже.
- **Бюджет:** ориентировочно 5–15 тыс. ₽/мес при 1 утренней сессии в день.
- **Когда подходит:** прототипирование, нерегулярные задачи, сложные интерфейсы.

### Вариант 2. Browserbase + Stagehand (или аналог)
- **Что это:** платформа headless-браузеров с AI-обёрткой. Stagehand даёт декларативный API ("найди кнопку 'Скачать'", "введи текст в поиск").
- **+ плюсы:**
  - Стабильнее, чем Computer Use (не "видит экран", а работает через DOM).
  - Дешевле в эксплуатации.
  - Можно параллелить (10 площадок в 10 потоков).
- **− минусы:**
  - Нужно немного программирования (TypeScript/Python).
  - Капча — сложнее.
- **Бюджет:** ~$99/мес (≈9 тыс. ₽) starter + плата за LLM (≈3–5 тыс. ₽/мес).
- **Когда подходит:** когда сценарий устаканился и надо стабильно крутить каждый день.

### Вариант 3. Playwright + LLM (самописное)
- **Что это:** обычный Playwright (бесплатный браузерный автомат) + Claude/GPT для анализа найденного.
- **+ плюсы:**
  - Бесплатно по инфраструктуре.
  - Полный контроль.
  - Можно крутить на твоём же Mac или дешёвом VPS (500 ₽/мес).
- **− минусы:**
  - **Нужно программирование** (Python/Node, ~неделя работы или подрядчик).
  - При смене вёрстки на площадке — ломается, надо чинить.
  - Капча и SSO решаются вручную.
- **Бюджет:** инфраструктура 500–1000 ₽/мес + LLM для анализа ~2–3 тыс. ₽/мес. Старт — оплата программисту 30–80 тыс. ₽ разово.
- **Когда подходит:** когда хочется минимальной ежемесячной стоимости и есть подрядчик.

### Вариант 4. Готовый сервис мониторинга тендеров (агрегатор)
- **Что это:** сервисы типа Seldon.Pro, Контур.Закупки, Тендерплан — агрегируют тендеры с площадок и дают веб-интерфейс / уведомления.
- **+ плюсы:**
  - Не надо ничего программировать. Подписка → получаешь.
  - Покрывают большинство площадок РФ.
- **− минусы:**
  - Не все коммерческие площадки покрыты.
  - Анализ "наш не наш" — на тебе, агрегатор не понимает специфику.
  - **Цена**: 3–10 тыс. ₽/мес базовая подписка, с фильтрами по регионам — выше.
- **Когда подходит:** если 80% тендеров — гос-закупки на крупных площадках. Если ниша = коммерческие на ДВ — может не закрыть.

## Рекомендация (обновлена 27.04.26)

Учитывая реальную цену агрегаторов (минимум 25К/мес) — **строим свой парсер сразу**.

### Архитектура

```
[Cron 1×/день]
    ↓
[5 скраперов Playwright по площадкам] → [Общая БД лотов]
    ↓                                          ↓
[Дедупликация по (площадка, № закупки)]   [Claude API: классификатор "наш/не наш"]
    ↓                                          ↓
[Топ-N лотов "наш"] → [Telegram утром Роману] + [Markdown в vault: 06_Builder/Tenders_<date>.md]
```

### Стек

- **Playwright** (Python или Node) — браузерный автомат для каждой площадки.
- **Claude API** (Sonnet 4.6) — анализ "наш / не наш" по нашим критериям.
- **SQLite** (для старта) или **Postgres** (если разрастётся) — БД лотов.
- **VPS** (например, Beget/Selectel ~500 ₽/мес) — крутится скрипт.
- **Telegram Bot** — у нас уже есть, переиспользуем.

### Ступень 1 (минимально жизнеспособная версия — MVP)

- 1–2 площадки (Сбер-А + РТС-тендер) — самые объёмные по госзакупкам.
- Простой фильтр по ключевым словам в названии лота ("перевозка", "экспедирование", "транспорт", "логистика").
- Без Claude-анализа на первой итерации — просто список найденного.
- Цель: за 1 неделю работы получить ежедневную сводку лотов в Telegram.
- Бюджет: 30–40 тыс ₽ подрядчику + 1 тыс ₽/мес VPS.

### Ступень 2 (через 2–3 недели после MVP)

- Добавляем оставшиеся 3 площадки (Roseltorg, B2B-Center, Тендерпро).
- Добавляем **Claude-классификатор**: каждый найденный лот сначала прогоняется по правилам ("наш"/"не наш"/"под вопросом"), потом сортируется.
- Добавляем извлечение ключевых данных из ТЗ (бюджет, сроки, требования к перевозчику).
- Бюджет: ещё 30–40 тыс ₽ подрядчику + 5–7 тыс ₽/мес LLM.

### Ступень 3 (через 1–2 месяца, опционально)

- Машинное обучение на наших же решениях ("этот взяли — этот нет") — модель уточняется по фидбеку.
- Полуавтоматическая подача заявки (с ручным "ОК" перед отправкой).

## Reference Case — мультимодал УСГ (победный)

Единственный выигранный тендер, на который равняемся:

- **Заказчик**: Усть-Среднеканская ГЭС-строй (УСГ).
- **Маршрут**: Магаданская область → Приморский край.
- **Схема**: мультимодальная авто-море-авто.
- **Сумма**: ~6 000 000 ₽.
- **Результат**: отработали блестяще (отзыв сохранить, попросить рекомендательное письмо — ценнейший актив для будущих заявок).
