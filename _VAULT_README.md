# AI-OS Vault — Операционная система собственника

Это база памяти твоей AI-системы управления. Obsidian открывает её как "хранилище" (vault).

## Зачем эта папка

Vault выполняет **две функции одновременно**, и обе важны:

1. **Второй мозг собственника (личная память).** Сюда сбрасывается всё, что не должно "вылететь из головы": обещания людям, мысли, идеи, договорённости, куда что положил, о чём договорился по телефону. Главное правило — **капчур должен быть максимально ленивым**: открыл дневник, записал в одну строку, закрыл. Структурирует потом Memory Curator, не ты.
2. **Долговременная память системы управления.** SOP, правила, решения, шаблоны, карточки машин/водителей/клиентов, журналы рейсов и инцидентов. Это то, что делает будущие решения умнее и дешевле.

**Что НЕ хранить:** мусор без выводов, сырые дампы переписок без пометки "важно", случайные скриншоты, дубли.

## Как открыть
1. Установи Obsidian (https://obsidian.md).
2. Нажми **Open folder as vault** и выбери эту папку `AI_OS_Vault`.
3. Всё. Никакой синхронизации пока не нужно. Позже можно подключить iCloud / Google Drive / Obsidian Sync.

## Правила работы с памятью

1. **Всё новое — в `90_Inbox/`**. Не пытайся сразу положить в правильную папку.
2. **Раз в неделю разбираем Inbox** (роль Memory Curator, в понедельник вместе со мной).
3. **В память попадает только полезное**: правила, решения, SOP, шаблоны, повторяющиеся ошибки, статусы, принципы, ключевые факты о людях и процессах.
4. **Черновики переписок, сырые дампы, скриншоты без выводов — в память не кладём.**
5. **У каждой заметки — короткое имя и назначение в первой строке**. Если через 3 месяца не ясно, о чём файл, — он бесполезен.

## Структура папок

```
00_Dashboard/   Главный пульт, еженедельный обзор
01_Operations/  Транспорт, машины, водители, рейсы (главное OPERATE-направление)
02_Sales/       Лиды, клиенты, воронка
03_Finance/     Платежи, налоги, лизинги, платёжный календарь
04_Import/      Сделки по импорту авто и спецтехники
05_Content/     Идеи, кейсы, публикации
06_Builder/     Сайты, боты, автоматизации
10_People/      Команда, контрагенты, клиенты
20_SOP/         Регламенты и повторяемые процедуры
30_Decisions/   Журнал ключевых решений (почему так, а не иначе)
40_Templates/   Базовые шаблоны (пока живут внутри README соответствующих папок)
90_Inbox/       Всё новое сначала сюда
99_System/      Роли агентов, правила маршрутизации, правила памяти
```

## Ритм работы

- **Каждый день (5 минут):** `01_Operations/Dispatch_Daily.md` — фотография парка на сегодня.
- **Каждый понедельник (20–30 минут):** `00_Dashboard/Weekly_Review.md` — пульт по шести контурам.
- **Раз в месяц (30 минут):** разбор `90_Inbox/`, обновление SOP, журнал решений.

## Текущий статус системы

- Этап: **старт, версия 0.1**.
- Активный контур OPERATE: **транспорт / операционка** (первый приоритет).
- Роли: пока все играю я; Memory Curator и Chief of Staff — в виде AI-ассистента.
- BUILD: сначала наполняем Operations реальными данными, потом превращаем в SOP и автоматизацию.
