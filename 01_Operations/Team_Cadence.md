# Ритм команды (Operations)

> Как собственник общается с операционной командой. Правило: собственник говорит с РОПом, РОП говорит с логистами. Прямой контакт с логистами — только по явному эскалейту или по желанию собственника, но не как норма.

## Пирамида

```
Собственник
    │
   РОП
    │
Логисты / Диспетчеры
    │
Водители
```

## Дневной цикл

- **09:00–09:30 — логисты** (внутри команды РОПа): планирование дня, расстановка машин, распределение заявок.
- **~10:00 — РОП → собственник:** короткий отчёт по состоянию парка (см. `10_People/Rop_Management.md`).
- **17:00–17:30 — логисты:** итоги дня, проблемы, вопросы на завтра.
- **Вечер — РОП:** короткая вечерняя сводка собственнику, только если что-то важное.

## Недельный цикл

- **Пн, 10:00 — Dispatch Huddle** (РОП + логисты, 15 минут): приоритеты недели, ключевые клиенты, проблемные машины.
- **Пт, 15:00 — Weekly Report** (РОП → собственник, письменно): формат в `10_People/Rop_Management.md`.
- **Пн, 19:00 — Weekly Review собственника** (в одиночку, с Chief of Staff / AI): `00_Dashboard/Weekly_Review.md`.

## Месячный цикл

- **Первая среда — 1-on-1** (собственник + РОП, 60 мин).
- **Вторая среда — Team Review** (РОП + логисты, 45 мин): цифры, процессы, конфликты, улучшения.
- **Последняя пятница — Финансовое закрытие** (совместно с бухгалтерией).

## Принципы эскалации

Что РОП **обязана** немедленно эскалировать собственнику:
- ДТП / серьёзная поломка, простой > 2 дней.
- Конфликт с ключевым клиентом (список — в `02_Sales/Key_Clients.md`).
- Любой финансовый риск > X руб. (порог определить вместе).
- Увольнение / инцидент с водителем.
- Претензия от контрагента / суда / налоговой.

Всё остальное — решает сама, сообщает в еженедельном отчёте.

## Где живут заметки с ритма

- `10_People/Rop_Weekly/YYYY-WW.md` — еженедельные отчёты РОПа.
- `01_Operations/Dispatch_Daily.md` — ежедневные снимки парка.
- `01_Operations/Incidents_Log.md` — журнал инцидентов.
- `30_Decisions/` — решения, которые были приняты в 1-on-1.
